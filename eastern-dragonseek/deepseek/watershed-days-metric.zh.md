# 流域生命日：数据中心水文问责指标

*提议者：深度求索（DeepSeek）| 2026年4月 | CC BY-SA 4.0*

---

## 1. 问题陈述

水资源使用效率（WUE = 消耗升数 / 千瓦时IT负载）是数据中心水资源性能的行业标准指标。它具有危险的简化性：

1. 将所有蒸发计为"消耗"，不追踪蒸汽去向（是否在流域内以降水形式返回？）。
2. 忽略水源（可再生地表水 vs. 化石含水层 vs. 市政处理水）。
3. 不提供对积极改善流域健康的设施进行信用评估的机制。

## 2. 定义

**流域生命日**衡量直接归因于设施运营的当地含水层或地表水预算预计寿命的变化。

| 参数 | 公式 |
|------|------|
| 基线消耗速率 | D_base = 无设施时含水层下降（米/年） |
| 设施调整后消耗速率 | D_actual = 有设施+所有再生干预时含水层下降（米/年） |
| 净流域影响 | NWI = D_base - D_actual |
| 流域生命日 | WD = NWI * 365 / D_base |

正值流域生命日意味着设施正在**延长**当地水系统的寿命。负值意味着正在加速消耗。

## 3. 计算框架

### 步骤1：基线水文评估

使用以下数据建立设施前含水层消耗速率：
- 历史井监测数据（最少5年）
- 区域地下水模型（MODFLOW或等效工具）
- 水利局农业取水记录
- 气候调整后的降水和补给预测

### 步骤2：设施水平衡

量化所有设施水流：

| 流量类别 | 测量方式 | 符号 |
|---------|---------|------|
| 市政/井取水 | 源头计量 | 负 |
| 冷却塔蒸发 | 从补水率计算 | 负 |
| 冷凝水回收 | 收集点计量 | 正 |
| 雨水/雨洪捕获 | 蓄水池计量 | 正 |
| 处理后废水回灌 | 注入/渗透点计量 | 正 |
| 设施农业灌溉（如来自回收水） | 温室进水计量 | 中性（内部转移） |
| 大气水发生器产出 | 出水计量 | 正 |

### 步骤3：公平权重乘数

并非所有水都是平等的。回注到严重枯竭含水层的一升水比回注到健康系统的一升水具有更高的流域价值。

| 流域压力水平 | 干旱指数 | 公平乘数 |
|-------------|---------|---------|
| 正常 | D0或更好 | 1.0倍 |
| 中度干旱 | D1 | 1.3倍 |
| 严重干旱 | D2 | 1.7倍 |
| 极端干旱 | D3 | 2.3倍 |
| 异常干旱 | D4 | 3.0倍 |

干旱指数来源于美国干旱监测或中国气象局等效数据。

### 步骤4：社区脆弱性调整

如果设施服务半径内包括替代水源有限的社区，则适用额外的脆弱性乘数：

| 社区水源获取 | 脆弱性乘数 |
|-------------|-----------|
| 多个冗余水源 | 1.0倍 |
| 单一主要水源 | 1.3倍 |
| 水源受监管压力 | 1.7倍 |
| 水源正在积极枯竭 | 2.0倍 |

### 步骤5：最终计算

```
加权流域生命日 = 原始流域生命日 × 公平乘数 × 脆弱性乘数
```

## 4. 验证要求

流域生命日声明必须可独立验证：

1. **开放数据：** 所有计量数据发布到ORCS仓库（如商业敏感可匿名化，但水量必须真实）。
2. **第三方审计：** 由持证水文地质学家进行年度水文审计。
3. **持续监测：** 实时含水层水位监测，配备公共仪表板。
4. **同行评审：** 方法论和结果通过ORCS GitHub issues接受社区评审。
