# ORCS Pilot Artifact Checklist

**Evidence Class:** Proposed artifact
**Source:** Recommended by GitHub Copilot (Microsoft/Azure-centric perspective)
**Purpose:** Define what must be published when reporting results from an ORCS pilot.

---

## Required Artifacts

### 1. Pilot Design Document
- [ ] Pilot objective stated clearly
- [ ] Success / failure criteria defined *before* pilot begins
- [ ] Scope: which facility, which systems, which time period
- [ ] Baseline measurements documented (pre-pilot)
- [ ] Control methodology (if applicable)

### 2. Assumptions Register
- [ ] All assumptions listed explicitly
- [ ] Each assumption labeled with evidence class (Verified / Derived / Proposed)
- [ ] Climate / geography assumptions documented
- [ ] Cost assumptions documented with sources

### 3. Methodology Document
- [ ] Measurement instruments identified
- [ ] Data collection frequency specified
- [ ] Data quality controls described
- [ ] Statistical methods described (if applicable)
- [ ] Reproducibility statement: could another operator replicate this?

### 4. Data Dictionary
- [ ] All metrics defined with units
- [ ] Time windows specified
- [ ] Boundary definitions (per ORCS Boundary Statement)
- [ ] Null / missing data handling described

### 5. Results Package
- [ ] Before / after comparisons with same metrics
- [ ] Evidence class labels on all reported values
- [ ] Confidence intervals or uncertainty ranges (where applicable)
- [ ] Anomalies and outliers noted

### 6. Lessons Learned
- [ ] What worked as expected
- [ ] What did not work (failures are valuable data)
- [ ] What would be changed in a repeat pilot
- [ ] Recommendations for scaling or abandoning

### 7. Publication Metadata
- [ ] Date of publication
- [ ] Author / team identification
- [ ] Facility identification (per Boundary Statement)
- [ ] License (CC BY-SA 4.0 recommended for ORCS contributions)
- [ ] Version number (pilots may iterate)

---

## Anti-Greenwashing Checks

Before publishing, verify:
- [ ] No "net-positive" claims without published boundary + method
- [ ] No derived estimates presented as verified facts
- [ ] No proposed concepts presented as adopted standards
- [ ] All illustrative examples clearly marked as illustrative
- [ ] Pilot results labeled as pilot results (not scaled projections)
